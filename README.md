# SmartMatching

`SmartMatching` сравнивает две генеалогии в формате `import.json` и возвращает одну лучшую пару совпавших персон.

## Что считается входом

- `tree1` и `tree2` — это **два отдельных дерева**, каждое в формате JSON-массива персон (структура как в `import.json`).
- Поле `treeId` у всех персон внутри одного входного массива должно быть одинаковым.
- Вход можно передавать как строку JSON или как уже распарсенный `list[dict]`.

## Что возвращает алгоритм

- Если найдено совпадение со `score >= threshold`, возвращается `dict`:
  - `score`
  - `tree1.id`, `tree1.personId`, `tree1.person`
  - `tree2.id`, `tree2.personId`, `tree2.person`
- Если валидного совпадения нет, возвращается пустой словарь `{}`.

## Методология сравнения

Сравнение выполняется по всем парам персон из двух деревьев (`N x M`).  
Для каждой пары рассчитывается итоговый `score` как взвешенное среднее по полям.

| Поле | Как сравнивается | Балл при отсутствии данных | Вес | Особенности |
|---|---|---:|---:|---|
| `surname` (фамилия) | `rapidfuzz.token_sort_ratio` после нормализации строки | 70 | 0.28 | Базовый идентификатор персоны |
| `name` (имя) | `rapidfuzz.token_sort_ratio` после нормализации строки | 70 | 0.24 | Базовый идентификатор персоны |
| `middleName` (отчество) | `rapidfuzz.token_sort_ratio` | 70 | 0.10 | Учитывается, если заполнено хотя бы у одной персоны |
| `birthdate` (дата рождения) | Сравнение диапазонов дат (`YYYY`, `YYYY-MM`, `YYYY-MM-DD`) | 50 | 0.28 | 100 при пересечении диапазонов, 70 при разнице по году <= 1 |
| `birthplace` (место рождения) | Токенизация + триграммы + коэффициент Жаккара | 50 | 0.10 | Удаляются служебные топонимы (`г`, `обл`, `район` и т.д.) |
| `maidenName` (девичья фамилия) | `rapidfuzz.token_sort_ratio` | 70 | 0.12 | Учитывается **только если есть в обеих** записях |

## Логика выбора лучшего совпадения

1. Для каждой пары персон между деревьями считается `score`.
2. Пары ниже `threshold` отбрасываются.
3. Из оставшихся выбирается одна пара с максимальным `score`.

## Быстрый тестовый запуск

Скрипт для тестирования `run_match_test.py`:

```bash
python3 run_match_test.py --tree1 ./import.json --tree2 ./import.json
```

С порогом:

```bash
python3 run_match_test.py --tree1 ./import.json --tree2 ./import.json --threshold 90
```
